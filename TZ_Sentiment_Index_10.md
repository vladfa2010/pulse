## 1. Общее описание

Интерактивный виджет «Индекс настроения» — игровая механика, где пользователь «оплачивает" доступ к графику коллективного настроения рынка своим голосом. Голосование происходит в **персональном 30-минутном окне** каждого пользователя. Результат влияет на общий индекс в реальном времени. Для анонимных пользователей показана часть графика с blur-оверлеем — мотивация к авторизации.

**Ключевое изменение v2:** голосование происходит в режиме **Blind Vote** — пользователь не видит текущий индекс и распределение голосов до момента голосования. Это исключает herd bias и обеспечивает честность данных.

**Режим работы:** 24 часа / 7 дней. Торговая сессия МосБиржи (10:00–19:00 МСК) выделена на графике визуально. IMOEX за пределами сессии показывается как flat line (последнее актуальное значение). Голосование настроения работает круглосуточно — инвесторы оценивают перспективы и вне сессии.

Поверх индекса настроения накладывается график индекса МосБиржи (IMOEX) для визуальной корреляции.

Страница состоит из трёх уровней:
1. **Основная карточка** — график + состояния (S0/S1/S2)
2. **Панели метрик** — личная статистика + сообщество
3. **Нижние блоки** — доска лидеров, достижения, история голосов

## 2. Игровая механика и UX-логика

### 2.1. Три состояния блока

| Состояние | Условие | Что видит пользователь |
|-----------|---------|------------------------|
| **S0: Аноним** | Нет сессии / не залогинен | Левая половина графика видна чётко. Правая половина под **тяжёлым Liquid Glass blur** (50% ширины). Внутри blur-зоны: иконка замка, текст «Актуальная динамика скрыта», подзаголовок, кнопка «Войти в аккаунт». Метрики и нижние блоки размыты и полупрозрачные. Текущий индекс виден. |
| **S1: Активный доступ** | Залогинен + проголосовал + прошло ≤ 30 мин с момента **его** последнего голоса | Полный чёткий график. Внизу карточки — **персональный** таймер обратного отсчёта. Зелёный пульсирующий индикатор. Последние 5 минут таймер краснеет. График обновляется в реальном времени (SSE). Метрики и нижние блоки полностью видны. Текущий индекс виден. |
| **S2: Ожидание голоса** | Залогинен + прошло > 30 мин с момента **его** последнего голоса (или никогда не голосовал) | Весь график под **экстремальном blur + затемнением**. Поверх — три кнопки голосования. **Текущее значение индекса скрыто** (blind vote). Метрики и нижние блоки видны, но график заблокирован до голоса. |

**Ключевой принцип:** у каждого пользователя свой персональный таймер. Один пользователь может быть в S1, другой — в S2, третий — вообще не голосовал сегодня. График при этом один общий и обновляется для всех в реальном времени.

### 2.2. Blind Vote (голосование вслепую)

**Проблема:** если пользователь видит текущий индекс перед голосованием, он может голосовать не по убеждению, а ради:
- Ачивки «в синхроне»
- Конфетти и зелёного тоста
- Повышения % синхрона в leaderboard

Это **self-reinforcing bias** — чем выше индекс, тем больше позитивных голосов ради синхрона. Индекс перестаёт отражать настроение и начинает отражать желание получить награду.

**Решение:**
- В состоянии S2 пользователь **не видит** текущий индекс (блок «Текущий индекс» скрыт blur)
- Пользователь **не видит** распределение голосов (progress bars в community metrics видны, но не обновляются в реальном времени для S2 — показываются данные на момент загрузки страницы)
- После голоса → S1, график открывается, показывается результат, sync/contrarian статус, конфетти (если в синхроне)

### 2.3. Голосование

#### 2.3.1. Персональные 30-минутные окна

- **Нет общей нарезки времени** для всех пользователей
- Каждый пользователь голосует в **своём собственном ритме**
- **Отсчёт:** от времени последнего голосования конкретного пользователя
- **Формула:** `next_vote_at = last_vote_at + 30 минут`
- Пользователь может проголосовать в любой момент после истечения его персонального окна
- Это обеспечивает **постоянную динамику графика** — голоса поступают асинхронно, график живёт непрерывно

#### 2.3.2. Параметры голоса

- **Значения:** `+1` (позитив), `0` (нейтрально), `-1` (негатив)
- **Кулдаун:** персональные 30 минут с момента последнего голоса данного пользователя
- **Тикеры:** автоматически подтягиваются из портфеля пользователя из существующей БД
- **Обновление графика:** после голоса линия индекса мгновенно пересчитывается и анимируется

### 2.4. Фидбек после голоса (Toast + Салют)

| Условие | Сообщение | Визуальный эффект |
|---------|-----------|-------------------|
| sign(мой голос) == sign(индекса) | «Вы в синхроне с настроением сообщества 🔥» | **SVG-конфетти (салют)** — 24 частицы, зелёная/золотая гамма, 1.8 сек. Toast с зелёной подсветкой |
| sign(мой голос) ≠ sign(индекса) | «Ваше мнение отличается — вы мыслите вне рамок 🧠» | Toast с фиолетовой подсветкой (контрарианство) |
| Голос = 0, индекс ≠ 0 | «Вы держите баланс ⚖️» | Обычный toast |

### 2.5. Формула индекса

**Кумулятивная сумма за текущие 24 часа (00:00–23:59 МСК):**

```
Index(t) = Σ vote_i   (i = 1..N, за последние 24 часа)
```

- Каждый день индекс стартует с **0** в 00:00 МСК
- Позитивный голос: `+1`, Негативный: `-1`, Нейтральный: `0`
- Линия строится по временным меткам голосований (асинхронно)
- **Нет нормализации по объёму** — для MVP оставляем кумулятивную сумму. В будущем: добавить относительный индекс (`(positive - negative) / total_votes * 100`)

### 2.6. IMOEX (индекс МосБиржи)

- Отображается как **второй график** (жёлтая пунктирная линия)
- Данные подтягиваются из существующего API (MOEX AlgoPACK / Finnhub / T-Bank API)
- **Торговая сессия:** 10:00–19:00 МСК выделена на графике полупрозрачной зоной (`rgba(245, 158, 11, 0.03)`)
- **За пределами сессии:** IMOEX показывается как flat line (последнее актуальное значение повторяется)
- Ось Y справа, масштаб независимый от индекса настроения
- Границы сессии (10:00 и 19:00) отмечены пунктирными линиями

### 2.7. 24-часовой режим

- Окно «сегодня» = 24 часа (00:00–23:59 МСК)
- `vote_count_today` сбрасывается в **00:00 МСК** (cron)
- `streak_days` проверяется в **00:00 МСК** (голосовал ли пользователь вчера)
- Голосование доступно круглосуточно — инвесторы оценивают перспективы и вне сессии
- На графике: торговая сессия визуально отделена, но индекс настроения непрерывен

## 3. Дизайн-система

### 3.1. Общая концепция: Liquid Glass

- **Фон карточки:** `rgba(255,255,255,0.025)` + `backdrop-filter: blur(40px) saturate(1.2)`
- **Границы:** `rgba(255,255,255,0.08)`, тонкие, полупрозрачные
- **Внутренние тени:** `inset 0 1px 1px rgba(255,255,255,0.05)`
- **Внешние тени:** глубокие, мягкие (`0 25px 50px -12px rgba(0,0,0,0.5)`)

### 3.2. Цветовая палитра

| Роль | Цвет | CSS |
|------|------|-----|
| Фон страницы | Глубокий тёмно-синий | `#0b0f19` |
| Позитив | Тёмный изумруд | `#10b981` |
| Негатив | Тёмный красный | `#ef4444` |
| Нейтраль | Платиновый | `#9ca3af` |
| IMOEX | Янтарный | `#f59e0b` |
| Золото (лидеры) | Янтарный | `#fbbf24` |
| Контрарианец | Фиолетовый | `#a855f7` |
| Текст основной | Белый с серым | `#f1f5f9` |
| Текст вторичный | Серо-голубой | `#94a3b8` |

### 3.3. График настроения

- **Линия:** 2.5px, tension 0.4
- **Цвет по сегментам:** зелёный выше 0, красный ниже 0, серый на 0
- **Заливка:** динамический градиент — зелёный если последнее значение > 0, красный если < 0, серый если = 0
- **Точки:** скрыты, появляются при hover (6px, белая заливка)
- **Зона сессии:** полупрозрачный фон `rgba(245, 158, 11, 0.03)` между 10:00 и 19:00

### 3.4. S0 — Blur для анонимов (правая половина)

- **Ширина:** 50% справа
- **Blur:** `blur(40px) saturate(0.5) brightness(0.6)`
- **Фон:** многослойный градиент + SVG noise (фрактальный шум, opacity 6%)
- **Граница:** анимированный shimmer — светящаяся вертикальная полоска
- **Внутренние блики:** `radial-gradient` с белым/зелёным/красным glow

### 3.5. S2 — Blur для голосования (полный) + Blind Vote

- **Canvas blur:** `filter: blur(12px) brightness(0.4)`
- **Overlay blur:** `blur(60px) saturate(0.2) brightness(0.15)`
- **Дополнительный слой:** псевдоэлемент `::before` с `blur(30px)` + `rgba(11,15,25,0.4)`
- **Фон:** `linear-gradient(180deg, rgba(11,15,25,0.85), rgba(11,15,25,0.97))` + noise
- **Внутренняя тень:** `inset 0 0 80px rgba(0,0,0,0.6)`
- **Скрытие индекса:** блок «Текущий индекс» скрыт через `opacity: 0; filter: blur(8px)`
- **Уведомление:** текст «🎯 Голосование вслепую: вы не видите текущий индекс до голоса»

### 3.6. Таймер (S1) — Персональный

- **Шрифт:** Inter, monospace, tabular-nums
- **Цвет:** зелёный (`#10b981`), последние 5 минут — красный (`#ef4444`) + glow
- **Индикатор:** пульсирующая зелёная точка (8px, `animation: pulse 2s infinite`)
- **Логика:** отсчитывает `next_vote_at - now` для текущего пользователя

### 3.7. Toast + Салют

- **Позиция:** fixed, bottom-center
- **Анимация:** `fadeInUp` (0.4s) → hold 3s → `fadeOut` (0.4s)
- **Салют:** 24 SVG-частицы, зелёная/золотая гамма, glow-вспышка, 1.8 сек
- **Синхрон-стиль:** зелёная рамка + glow-тень
- **Контрарианец-стиль:** фиолетовая рамка + glow-тень (без конфетти, но с уважительным сообщением)

### 3.8. Панели метрик

**Layout (desktop):**
```
┌─────────────────────────────────────────────┐
│  📈 Топ тикеры (grid-column: 1 / -1)          │
├─────────────────────┬───────────────────────┤
│  🌍 Сообщество      │  👤 Моя статистика    │
├─────────────────────────────────────────────┤
│  🎖 Мои достижения (grid-column: 1 / -1)      │
└─────────────────────────────────────────────┘
```

**Контейнер:** `display: grid; grid-template-columns: 1fr 1fr; gap: 20px` под основной карточкой.  
**Топ тикеры:** занимает обе колонки (`grid-column: 1 / -1`).  
**Сообщество + Моя статистика:** в одну линию, по колонке каждый.  
**Мои достижения:** занимает обе колонки (`grid-column: 1 / -1`), 12 бейджей в сетке `3×4`.  
**На mobile (< 768px):** все четыре блока в одну колонку, gap: 10px, padding уменьшен.

**S0:** метрики `blur(4px) opacity(0.3)` — не отвлекают от призыва войти.

**S1/S2:** метрики чёткие, обновляются после голоса.

**Тултипы:** каждый пункт метрик (и личных, и сообщества) имеет всплывающую подсказку при наведении, объясняющую смысл метрики.

### 3.9. Нижние блоки (3 карточки)

**Контейнер:** `display: grid; grid-template-columns: 1fr 1fr 1fr; gap: 20px` под метриками. На mobile (< 768px) — 1 колонка.

**S0:** все три блока размыты (`blur(4px) opacity(0.3)`).

**S1/S2:** блоки полностью видны и интерактивны.

**Тултипы на заголовках:**

## 5. Нижние блоки (3 карточки)

### 5.2. Топ-10 лидеров (Leaderboard)

**Заголовок:** «🏆 Топ-10 лидеров»

**Тултип на заголовке:** при наведении на текст «Топ-10 лидеров» — «Сортировка по проценту синхрона (убывание), при равенстве — по серии дней. Минимум 10 голосов для попадания.»

**Позиция:** левая нижняя карточка (1 из 2)

**Содержимое:** список из 10 строк. Каждая строка:
- **Ранг:** круг 24×24px с номером. 1-е место — золотой градиент (`#fbbf24` → `#f59e0b`) + glow. 2-е — серебряный. 3-е — бронзовый. 4-10 — полупрозрачный серый фон.
- **Никнейм:** 13px, medium, белый. Анонимизация — только публичные никнеймы, без персональных данных.
- **Серия:** огненный эмодзи 🔥 + число дней (11px, жёлтый).
- **Синхрон:** процент (12px, зелёный, tabular-nums, bold).

**Сортировка:** по проценту синхрона (убывание), при равенстве — по streak (убывание).

**Обновление:** раз в 5 минут или при SSE-событии.

**SQL (пример):**
```sql
SELECT 
    u.nickname,
    w.sync_count * 100.0 / w.total_votes_count as sync_rate,
    w.streak_days
FROM sentiment_user_windows w
JOIN users u ON u.id = w.user_id
WHERE w.total_votes_count > 10  -- минимум 10 голосов для попадания
ORDER BY sync_rate DESC, streak_days DESC
LIMIT 10;
```

### 5.3. Мои достижения (Badges) — v2.1

**Заголовок:** «🎖 Мои достижения»

**Позиция:** центральная нижняя карточка (2 из 3)

**Содержимое:** сетка `3×4` (12 бейджей). Каждый бейдж — квадратная карточка:
- **Иконка:** эмодзи 24px (центрирован)
- **Название:** 10px, uppercase, серый (заблокирован) / белый (разблокирован)
- **Состояние:** `locked` (opacity 0.35, grayscale) или `unlocked` (полная яркость, цветная подсветка границы)
- **Тултип:** при наведении на каждый бейдж показывается детальное условие разблокировки

**Список бейджей (12 шт):**

| Бейдж | Иконка | Условие разблокировки | Сложность проверки | Тултип |
|-------|--------|----------------------|-------------------|--------|
| Первый голос | 🗳 | Проголосовать впервые | Мгновенно (INSERT) | «Проголосовать впервые. Начало пути инвестора-аналитика.» |
| 100 голосов | 💯 | Набрать 100 голосов всего | Мгновенно (COUNT) | «Набрать 100 голосов за всё время. Вы регулярно делитесь мнением с сообществом.» |
| 7 дней подряд | 🔥 | Серия 7 дней | Cron в 00:00 МСК | «Голосовать хотя бы раз в день 7 дней подряд. Дисциплина — основа трейдинга.» |
| Интуиция 80% | 🧠 | Синхрон > 80% (мин. 20 голосов) | Мгновенно (COUNT) | «Синхрон с настроением сообщества > 80%. Минимум 20 голосов. Вы чувствуете рынок.» |

### 7.4. Отложенные проверки бейджей (Cron) — v2

**Проблема:** «Контрарианец» и «Точный прогноз» требуют проверки условий через время после голоса (2ч и 1ч).

**Решение:** Cron-задача, запускаемая каждые 5 минут.

```python
# Псевдокод (Python/psycopg)
def check_delayed_badges():
    now = datetime.now(timezone.utc)

    # 1. Точный прогноз (проверяем голоса старше 1 часа, но не старше 3)
    votes = db.query("""
        SELECT id, user_id, vote_value, imoex_at_vote, created_at
        FROM sentiment_votes 
        WHERE check_status = 'pending' 
          AND created_at <= %(now)s - INTERVAL '1 hour'
          AND created_at > %(now)s - INTERVAL '3 hours'
    """, {"now": now})

    for vote in votes:
        imoex_1h = get_imoex_at(vote.created_at + timedelta(hours=1))

        accurate = False
        if vote.vote_value == 1 and imoex_1h > vote.imoex_at_vote:
            accurate = True
        elif vote.vote_value == -1 and imoex_1h < vote.imoex_at_vote:
            accurate = True
        elif vote.vote_value == 0 and abs(imoex_1h - vote.imoex_at_vote) < 0.1:
            accurate = True

        db.execute("""
            UPDATE sentiment_votes 
            SET imoex_after_1h = %s, check_status = 'checked'
            WHERE id = %s
        """, (imoex_1h, vote.id))

        if accurate:
            # Увеличиваем streak точных прогнозов
            db.execute("""
                UPDATE sentiment_user_windows
                SET forecast_streak = forecast_streak + 1,
                    max_forecast_streak = GREATEST(max_forecast_streak, forecast_streak + 1)
                WHERE user_id = %s
            """, (vote.user_id,))

            # Проверяем бейдж (5 подряд)
            window = db.query("""
                SELECT forecast_streak FROM sentiment_user_windows WHERE user_id = %s
            """, (vote.user_id,))[0]

            if window.forecast_streak >= 5 and 'accurate_forecast' not in get_badges(vote.user_id):
                award_badge(vote.user_id, 'accurate_forecast')
        else:
            # Сбрасываем streak
            db.execute("""
                UPDATE sentiment_user_windows SET forecast_streak = 0 WHERE user_id = %s
            """, (vote.user_id,))

    # 2. Контрарианец (проверяем голоса старше 2 часов)
    votes = db.query("""
        SELECT id, user_id, vote_value, index_at_vote, created_at
        FROM sentiment_votes 
        WHERE check_status = 'pending' 
          AND created_at <= %(now)s - INTERVAL '2 hours'
    """, {"now": now})

    for vote in votes:
        index_2h = get_index_at(vote.created_at + timedelta(hours=2))

        was_contrarian = sign(vote.vote_value) != sign(vote.index_at_vote)
        index_turned = sign(index_2h - vote.index_at_vote) == sign(vote.vote_value)

        db.execute("""
            UPDATE sentiment_votes 
            SET index_after_2h = %s, check_status = 'checked'
            WHERE id = %s
        """, (index_2h, vote.id))

        if was_contrarian and index_turned:
            db.execute("""
                UPDATE sentiment_user_windows 
                SET contrarian_count = contrarian_count + 1
                WHERE user_id = %s
            """, (vote.user_id,))

            if 'contrarian' not in get_badges(vote.user_id):
                award_badge(vote.user_id, 'contrarian')
```

**Альтернатива:** если cron недоступен на Render — использовать **lazy-проверку**: при каждом новом голосе пользователя проверять его старые голоса (старше 1ч/2ч). Но это пропустит неактивных пользователей.

**Рекомендация для MVP:** cron каждые 5 минут в отдельном lightweight-контейнере на Render.
